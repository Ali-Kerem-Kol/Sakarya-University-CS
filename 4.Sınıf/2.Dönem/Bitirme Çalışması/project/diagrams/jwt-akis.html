Giriş Akışı (Login Flow)
1
İstemci → POST /api/v1/auth/login isteği gönderir
(e-posta + şifre)
2
AuthController isteği karşılar ve AuthService'e yönlendirir
3
AuthService, UserAccountRepository ile kullanıcıyı bulur
BCrypt ile şifre doğrulaması yapar
4
JwtService, HMAC-SHA256 ile JWT belirteci oluşturur
Payload: { sub: email, role: ADMIN/USER, iss: "project", exp: 60dk }
5
İstemciye JWT belirteci ve kullanıcı bilgileri JSON olarak döner
Token, localStorage'a kaydedilir
Kimlik Doğrulama Akışı (Her İstek İçin)
6
İstemci, her API isteğinde Authorization: Bearer <token> başlığını ekler
(Axios interceptor otomatik ekler)
7
JwtAuthenticationFilter (OncePerRequestFilter), Authorization başlığını okur
Bearer şemasından token'ı çıkarır
8
JwtService.validateToken() çağrılır:
• İmza doğrulanır (HMAC-SHA256)
• Süre kontrolü yapılır (exp claim)
• Issuer doğrulanır
9
SecurityContext'e kimlik bilgileri yerleştirilir
UsernamePasswordAuthenticationToken oluşturulur
10
Spring Security yetkilendirme kurallarını uygular:
• /api/v1/admin/** → ADMIN rolü gerekli
• /api/v1/me/** → Giriş yapmış kullanıcı gerekli
• /api/v1/public/** → Herkese açık
Hata Durumları
!
Geçersiz/Süresi dolmuş token: 401 Unauthorized → RestAuthenticationEntryPoint → İstemci /login'e yönlendirir
!
Yetersiz yetki: 403 Forbidden → RestAccessDeniedHandler → Erişim engellenir
